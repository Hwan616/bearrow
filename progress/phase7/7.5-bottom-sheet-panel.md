## 한 일

### 패키지 설치
- `react-native-reanimated@^3.19.5` — @gorhom/bottom-sheet 필수 피어 의존성
  - v4 대신 v3 채택: v4는 `react-native-worklets` 피어 의존성이 Jest 환경에서 네이티브 모듈 로드를 시도하여 테스트 불가
- `@gorhom/bottom-sheet@^5.2.14` — 스냅 시트 컴포넌트

### babel.config.js
- `"react-native-reanimated/plugin"` 플러그인 추가 (reanimated 필수)

### package.json jest 설정
- `"setupFiles": ["react-native-reanimated/mock"]` 추가 → 테스트 환경에서 reanimated 네이티브 모듈 모킹

### AppBottomSheet (`src/ui/AppBottomSheet.tsx`)
- `snapPoints = ["50%", "90%"]` — 2단계 스냅
- `index={-1}` 초기 상태(닫힘), `visible` prop 변경 시 `snapToIndex(0)` / `close()` 호출
- `enablePanDownToClose` — 50% 미만 드래그 시 닫힘
- `onChange(index)` — index === -1일 때 `onClose()` 호출
- `BottomSheetView` 래핑으로 children 렌더

### AppSidePanel (`src/ui/AppSidePanel.tsx`)
- `visible=false` 시 null 반환 (언마운트)
- `visible=true` 시 우측 고정 절대 포지션 View 렌더
- `width` prop (기본 320pt), `position: absolute, right: 0`
- `borderLeftWidth`, `shadowColor/Offset/Opacity/Radius`, `elevation` 그림자 처리

### 테스트
- `AppBottomSheet.test.tsx` (5개): 렌더, children 포함, visible 전환, onChange(-1)→onClose, onChange(0)→onClose 미호출
- `AppSidePanel.test.tsx` (5개): visible=true 렌더, visible=false 미렌더, children 포함, false→true 전환, width prop 반영

## 주요 결정

- **reanimated v3 채택 (v4 아님)**: v4(`4.5.1`)는 `react-native-worklets` 피어 dep이 있고, 해당 패키지가 Jest 환경에서 네이티브 모듈(`NativeWorklets`)을 로드하려 해 테스트 전체가 실패함. `@gorhom/bottom-sheet@5.2.14`는 `>=3.16.0 || >=4.0.0-` 피어 dep을 지원하므로 v3.19.5가 완전히 호환됨.
- **App.tsx 미수정**: 현재 App.tsx의 투두/설정 Modal은 7.6(투두 시트), 7.7(설정 시트)에서 교체 예정. 7.5는 컨테이너 컴포넌트 제공에 집중.
- **AppSidePanel onClose prop**: 와이드 레이아웃의 닫힘 제어를 위해 선언 포함; 7.6/7.7에서 실제 활용.
- **@gorhom/bottom-sheet 완전 모킹**: 테스트에서 `@gorhom/bottom-sheet`를 전체 mock하여 네이티브 의존성 완전 회피. `__triggerOnChange` 헬퍼로 onChange 콜백 시뮬레이션.
