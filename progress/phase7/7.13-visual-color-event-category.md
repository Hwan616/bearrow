## 주요 결정

- **당월 외 완전 숨김**: 구글 캘린더처럼 기존에 회색으로 표시하던 방식에서, 아예 빈 셀만 남기는 방식으로 전환. 그리드 크기(42칸)는 유지하여 높이 고정.
- **강조색 제거**: 카테고리 색상이 유일한 색상 커스터마이징이 되므로, 시스템 전반 강조색은 더 이상 노출하지 않음. `useTheme`의 `accentColor`/`setAccentColor` 구현은 코드에 남아 있으나 UI에서만 숨김.
- **이벤트 카테고리 필수화**: UI에서 첫 번째 카테고리를 자동 선택하여 항상 하나 이상이 선택된 상태가 되도록 설계.

## 남은 과제 / 주의사항

- YearView `YEAR_ITEM_HEIGHT` 640이 실기기에서 실제 렌더 높이와 완전히 일치하는지 확인 필요.
- 이벤트가 카테고리 없이 저장된 기존 데이터는 카테고리 미표시 상태가 될 수 있음 (DB 마이그레이션 불필요, UI에서 `colors.accent.primary` fallback 유지).
