## 한 일

### YearView (`YearView.tsx`)
- `ScrollView` + `useYearData` 훅 제거 → `FlatList<number>` (연도 숫자 배열)로 교체
- ±10년 = 21개 항목 생성 (`generateYearList`)
- 각 항목(`YearItem`): 연도 타이틀 + 12개 미니 월 그리드 (항상 6행으로 패딩)
- ‹ › 연도 이동 버튼 삭제 → 세로 스크롤로 탐색
- `getItemLayout` + `YEAR_ITEM_HEIGHT=688` 으로 정확한 초기 스크롤 위치 계산
- `useYearData.ts` / `useYearData.test.ts` 미사용 파일 삭제

### MonthView (`MonthView.tsx`)
- 단일 `View` + 로컬 year/month 상태 제거 → `FlatList<YearMonth>` (49개 = ±24달)로 교체
- `generateMonthList` 헬퍼로 중앙 기준 ±24달 배열 생성
- `padTo6Weeks` 헬퍼: `buildMonthGrid` 결과를 항상 42칸(6주)으로 패딩 → 고정 `MONTH_ITEM_HEIGHT=608`
- 각 항목(`MonthItem`): 월 타이틀(탭 → `YearMonthPicker`) + 요일 헤더 + 6주 그리드
- `useMonthItems` 훅은 MonthItem 내부에서 각자 호출 (FlatList `windowSize=5` → 최대 5달 동시 활성)
- `YearMonthPicker` 유지 — 월 타이틀 탭 시 열리며, 선택 시 `scrollToIndex`로 이동
- ‹ › 월 이동 버튼 삭제 → 세로 스크롤로 탐색

### Day View 헤더 날짜 미갱신 버그 수정 (동일 세션)
- `itemVisiblePercentThreshold: 50` → `viewAreaCoveragePercentThreshold: 50`
- 1576px 아이템에 700px 뷰포트일 때 itemVisible threshold는 물리적으로 도달 불가

### 편집 화면 노치 아래 버튼 가림 수정 (동일 세션)
- `EventDetailSheet`, `EventForm`, `TodoForm` 세 컴포넌트에 `SafeAreaView` 추가

## 주요 결정

- **항상 6주 렌더**: 4주·5주 달도 6주로 패딩하여 `getItemLayout` 고정 높이 보장. 빈 셀은 `isCurrentMonth: false`로 흐리게 표시 (구글 캘린더 표준 방식).
- **`viewAreaCoveragePercentThreshold`**: DayView와 동일한 이유로 선택. 화면을 꽉 채우는 큰 항목에는 `itemVisiblePercentThreshold`가 작동 안 함.
- **useYearData 삭제**: YearView만 사용하던 훅으로, 재작성 후 완전 미사용.

## 남은 과제

- `getItemLayout` 높이 추정값(688, 608)이 실제 폰트 렌더 높이와 약간 다를 수 있어 초기 스크롤 위치가 몇 픽셀 어긋날 수 있음. 실기기 확인 권장.
- `windowSize=5` (MonthView)로 최대 5달이 동시 렌더. DB 쿼리 5개 병렬 — 성능 이슈 없으나 모니터링 권장.
