## 변경 요약

### 무엇을

**EventDetailSheet — 편집·삭제 추가:**
- 헤더 좌측: 빨간 "삭제" 버튼 → `Alert.alert` 확인 → `deleteEvent` → `onDeleted` 콜백(캘린더 갱신) → 시트 닫힘
- 본문 "편집" 버튼 → 중첩 Modal로 `EventForm(initialEvent=event)` 오픈 → 저장 후 `onSaved` 콜백(캘린더 갱신) → 시트 닫힘
- `Props`: `onSaved?: () => void`, `onDeleted?: () => void` 추가

**DayDetailPanel — 할일 인터랙션:**
- `TodoRow`: 원형 체크박스 클릭 → `toggleTodo` 직접 호출 후 로컬 `refresh()`, 텍스트 영역 클릭 → `onEdit` prop으로 편집 모달 오픈
- `onEditTodo?: (todo: Todo) => void` prop 추가

**useDayItems — refresh 지원:**
- `version` 상태 추가, `useEffect` deps에 포함
- `refresh: () => void` 반환값 추가

**App.tsx:**
- `DayDetailPanel`에 `onEditTodo={setEditingTodo}` 연결 (투두 탭의 편집 모달 공유)
- `EventDetailSheet`에 `onSaved`, `onDeleted` 연결 (캘린더 키 증가)

### 왜

이벤트 상세 화면에서 편집·삭제가 불가능해 생성 후 수정·삭제를 위해 앱을 재시작해야 하는 문제. DayDetailPanel의 할일은 읽기 전용으로만 표시되어 완료 토글조차 불가능한 문제.
