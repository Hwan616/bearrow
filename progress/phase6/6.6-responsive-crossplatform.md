## 변경 요약

### 무엇을

**App.tsx (전면 재작성):**

- `useWindowDimensions` 훅으로 창 너비를 실시간 추적
- **768pt 기준 2-레이아웃 전환:**
  - **컴팩트(≤767pt)**: 기존 스마트폰 레이아웃 (하단 탭 바 + FAB)
  - **와이드(≥768pt)**: 사이드바 레이아웃 (iPad · Mac · 데스크톱)
- **와이드 사이드바(280pt):**
  - 앱 이름 "BeArrow" 헤더
  - MonthView — 날짜 클릭 시 선택 날짜 갱신 + 캘린더 탭 자동 포커스
  - 내비게이션 항목(캘린더 / 할일 / 설정) — 동일 testID 유지
  - 탄력 여백(flex:1)으로 추가 버튼 하단 고정
  - 컨텍스트 추가 버튼 ("＋ 새 일정" / "＋ 새 할일") — 동일 testID 유지
- **와이드 메인 영역(flex:1):**
  - 캘린더 탭: DayDetailPanel 단독 (MonthView는 사이드바)
  - 할일 탭: TodoMiniCalendar + TodoList (컴팩트와 동일)
  - 설정 탭: SettingsScreen
- **Android Ripple** (`android_ripple`) FAB·탭 항목·사이드바 내비에 추가 → Android에서 터치 피드백 시각화
- `Modal presentationStyle="pageSheet"` 제거 → iOS 전용 속성 제거로 플랫폼 중립화
- 탭 바 반복을 `NAV_ITEMS` 배열로 선언해 코드 중복 제거
- FAB를 "설정 탭에서 숨김" 조건으로 단순화

### 왜

스마트폰 전용 하단 탭 레이아웃은 iPad나 MacBook 같은 넓은 화면에서 공간을 비효율적으로 사용하고, 시각적 일관성이 떨어진다. 또한 `presentationStyle="pageSheet"`와 같은 iOS 전용 속성은 Android·웹에서 다른 모달 외형을 초래한다.

## 주요 결정

- **분기점 768pt**: iPad mini(768pt)부터 가로로 충분한 공간이 생기는 기준점
- **사이드바에 MonthView 배치**: 캘린더와 할일 탭 모두에서 월 내비게이션을 항상 볼 수 있어 맥락 유지
- **와이드 캘린더 탭**: MonthView를 사이드바로 이동해 메인 영역이 DayDetailPanel에만 집중 (기존 컴팩트의 embedded ScrollView 레이아웃과 역할 분리)
- **testID 동일 유지**: 와이드 레이아웃에서도 `fab-add-event`, `fab-add-todo`, `tab-calendar` 등 기존 testID를 그대로 사용 → 스모크 테스트 무수정 통과
- **`makeStyles(colors)` 컴포넌트 내부 호출**: 색상 토큰을 스타일에 안전하게 반영
