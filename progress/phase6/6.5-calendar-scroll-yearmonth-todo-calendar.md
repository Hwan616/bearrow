## 변경 요약

### 무엇을

**YearMonthPicker.tsx (신규):**
- Modal 기반 연월 선택 팝업
- 상단: 연도 내비게이션 (‹ 2026년 ›)
- 하단: 3행 4열 월 버튼 그리드 (1월~12월)
- 현재 연월 강조(채움/테두리), MonthView·TodoMiniCalendar 공용

**MonthView.tsx (수정):**
- 월 제목 텍스트를 `Pressable`로 감싸 클릭 시 YearMonthPicker 오픈
- `pickerVisible` 상태 추가

**DayDetailPanel.tsx (수정):**
- `embedded?: boolean` prop 추가
- `embedded=true`: 내부 `ScrollView` 제거, `flex:1` 해제 → 부모 ScrollView가 스크롤 담당
- 공통 `listContent` JSX 변수로 분리해 코드 중복 최소화

**App.tsx — 캘린더 탭 통합 스크롤:**
- `<View>` → `<ScrollView contentContainerStyle={{ paddingBottom: 100 }}>` 교체
- `DayDetailPanel`에 `embedded` prop 전달
- 결과: MonthView 그리드 → 선택 날짜 일정·할일 목록이 하나의 스크롤로 이어짐

**TodoMiniCalendar.tsx (신규):**
- 투두 탭 상단에 표시되는 컴팩트 월 그리드
- 미완료 할일이 있는 날짜에 녹색 숫자 뱃지 표시 (`useMemo`로 날짜별 집계)
- 공휴일 색상 규칙 MonthView와 동일
- 날짜 변경 모드(`reschedulingTodo`)일 때: 황색 테두리 + "새 날짜를 선택하세요" 배너
- 현재 마감일 날짜는 주황 원으로 강조

**TodoItem.tsx (수정):**
- `onReschedule?: (todo: Todo) => void` prop 추가
- 날짜 변경 버튼(↗) — 달력 모드 진입 트리거

**TodoList.tsx (수정):**
- `onReschedule?: (todo: Todo) => void` prop 추가 → TodoItem으로 전달

**App.tsx — 투두 탭 미니 캘린더:**
- `reschedulingTodo` 상태 추가
- `allTodos = sections.flatMap(s => s.todos)` — 월별 집계용
- `handleReschedule(newDate)` — handleUpdate 호출 후 calendarKey 증가
- TodoMiniCalendar + TodoList를 `View flex:1` 안에 배치

### 왜

1. 캘린더 탭: MonthView와 DayDetailPanel이 별도 스크롤 영역으로 분리돼 있어 주류 캘린더 앱과 달리 자연스러운 스크롤 탐색이 불가능했다.
2. 연월 이동: 화살표만으로는 멀리 떨어진 달(예: 6개월 후)로 이동하기 번거로웠다.
3. 투두 탭: 할일 목록만 보여줘 날짜별 부하를 파악하기 어렵고, 마감일 변경을 위해 편집 모달을 열어야 했다.

## 주요 결정

- `DayDetailPanel`에 새 컴포넌트를 만들지 않고 `embedded` prop으로 확장 → 기존 코드 재사용
- 투두 미니 캘린더의 날짜 변경: 드래그 대신 버튼 클릭 → 달력 선택 2단계로 구현, 복잡한 제스처 없이 명확한 UX
- `allTodos`는 `sections.flatMap`으로 계산해 별도 API 호출 없음
- 셀 최소 높이 44pt 유지(UI-003 준수), count 뱃지는 높이 12px 플레이스홀더로 셀 높이 일관성 확보
