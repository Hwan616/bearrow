## 한 일

### iCalendar 파서 (`src/sync/icalParser.ts`)
- RFC 5545 iCalendar 형식 파싱: VEVENT → ParsedVEvent
- 지원 필드: UID, SUMMARY, DESCRIPTION, DTSTART(UTC·로컬·종일), DTEND, RRULE, LAST-MODIFIED, STATUS
- 줄 이어쓰기(line folding) 복원, 이스케이프 문자 복원 (`\n`, `\,`, `\;`, `\\`)
- CalDAV 응답 XML 헬퍼: `extractCalendarDataBlocks`, `extractHrefInTag`, `extractResponseHrefs`
- 단위 테스트 15개 (`src/sync/__tests__/icalParser.test.ts`)

### CalDAV 클라이언트 (`src/sync/caldavClient.ts`)
- iCloud CalDAV 연동 최소 구현
- 발견(discovery): GET `.well-known/caldav` → 리디렉션 후 서버 origin 획득
- PROPFIND: current-user-principal → calendar-home-set → 캘린더 목록
- REPORT (calendar-query): ±1년 범위 VEVENT 수집

### iCloud 동기화 (`src/sync/icloudCalendarSync.ts`)
- Apple ID + 앱 전용 암호를 `expo-secure-store`에 안전하게 저장·조회·삭제
- `syncICloud()`: 캘린더별 REPORT → icalParser → SQLite 적용 (upsert)
  - 로컬 `updated_at` vs. LAST-MODIFIED 비교로 최신 데이터 유지
  - `source = 'apple'` 으로 외부 이벤트 식별
- `verifyICloudConnection()`: 연결 전 자격 증명 사전 검증

## 주요 결정

- **앱 전용 암호**: Apple은 표준 Apple ID 암호로 CalDAV 직접 접근을 허용하지 않으므로, appleid.apple.com에서 발급한 앱 전용 암호를 사용한다.
- **단방향(읽기 전용)**: iCloud → BeArrow 가져오기만 구현. iCloud에 쓰기는 인증·충돌 해결 복잡도가 높아 향후 과제로 남김.
- **XML 라이브러리 미추가**: CalDAV 응답 구조가 예측 가능하므로 정규식 파싱으로 충분. 의존성 추가 없이 구현.
- **`source = 'apple'`**: 기존 스키마 enum 값("local" | "google" | "apple") 그대로 사용. DB 마이그레이션 불필요.
- **Google sync trigger**: 기존 `syncGoogleCalendar` 함수가 완전 구현되어 있으므로, 설정 화면에서 Supabase `session.provider_token`(Google OAuth 토큰)을 넘기는 것만 추가.

## 남은 과제 / 주의사항

- **Google Cloud Console 설정 필요** (초기 1회): Google Cloud 프로젝트 생성 → Google Calendar API 활성화 → OAuth 2.0 클라이언트 ID 생성 → Supabase Google 공급자 설정 → `.env`에 키 입력. 이후 "Google로 로그인" 버튼만으로 자동 스코프 포함.
- **iCloud 앱 전용 암호**: 사용자가 [appleid.apple.com](https://appleid.apple.com) 에서 직접 발급해야 함.
- **CalDAV 파티션 서버**: Apple은 iCloud 사용자를 `p01~p##-caldav.icloud.com` 파티션 서버로 라우팅한다. `fetch(redirect: "follow")` + `response.url`로 자동 처리.
- **iCloud → BeArrow 쓰기**: 미구현. 향후 Phase에서 CalDAV PUT/DELETE 추가 가능.
