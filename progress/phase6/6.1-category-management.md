# 6.1 카테고리 관리 완성 (투두)

| 항목 | 내용 |
|---|---|
| 날짜 | 2026-07-05 22:05 |

## 변경 요약

투두리스트의 카테고리 기능을 완성했다.

### 무엇을

- **카테고리 생성·편집·삭제 UI** (`CategoryForm`, `CategoryManager`) 신규 구현
  - `CategoryForm`: 이름(미입력 시 '카테고리' 기본값), 색상 팔레트, 미리보기
  - `CategoryManager`: 목록·편집·삭제 버튼, 중첩 Modal 구성
- **삭제 정책 강화** (`categories.ts`): 마지막 카테고리 삭제 불가, 삭제 시 할일·일정을 fallback 카테고리로 자동 재배정
- **기본 카테고리 자동 생성** (`ensureDefaultCategory`): 앱 초기화(`App.tsx`) 시 DB에 카테고리가 없으면 이름 '카테고리', 색상 `#2E5AAC`로 생성
- **할일 필수 카테고리** (`TodoForm`, `useTodos`): `categoryId` 타입을 `string | null` → `string`으로 변경, '없음' 칩 제거, 카테고리 로드 후 첫 번째를 기본값으로 자동 선택
- **설정 화면 개편** (`SettingsScreen`): '카테고리 색상' 인라인 UI → '카테고리 관리' 버튼 + Modal 방식으로 교체
- **useCategories 훅** 신규 추가: create/update/delete 핸들러와 자동 refresh 로직
- **테스트 갱신**: `deleteCategory` 기존 테스트(null 재배정) → 새 동작(fallback 재배정, 1개 오류)으로 전면 재작성, `ensureDefaultCategory` 테스트 추가, App 스모크 테스트 모킹 추가

### 왜

시뮬레이터 검증에서 카테고리 CRUD UI가 없고, 할일이 카테고리 없이 생성 가능한 문제 발견.
요구사항상 할일은 반드시 1개의 카테고리를 가져야 한다.
